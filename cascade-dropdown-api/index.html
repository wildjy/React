3-API Cascade단계마다 fetch

Cascade Dropdown — 단계별 독립 API 버전

대학을 고르면 계열 API(univCode), 계열을 고르면 학과 API(univCode + aiBdPart) 가 호출됩니다. 응답 DTO 는 hook 에서 { value, label } 로 변환되어 드롭다운에 들어갑니다.

API 응답 스키마

대학
GET /universities
univCode · univName
[{ "univCode": "0001",
   "univName": "서울대학교" }]
계열
GET /majorTypes?univCode
aiBdPartCode · aiBdPart
[{ "aiBdPartCode": "인문",
   "aiBdPart": "인문" }]
학과
GET /majors?univCode&aiBdPart
majorIdHsb · majorName
[{ "majorIdHsb": "12345",
   "majorName": "컴퓨터공학과" }]
1대학apiFetchUniversities()
    2계열apiFetchMajorTypes(univCode)
      3학과apiFetchMajors(univCode, aiBdPart)
        현재 selected (다음 API 로 보낼 코드들)
        {
  "univCode": "",
  "aiBdPart": "",
  "majorIdHsb": ""
}

        캐시 버전과 무엇이 다른가

        • •계열 선택 = 또 한 번의 fetch. 캐시 버전은 받아둔 데이터를 client 에서 필터했지만, 여기선 학과 목록을 서버에서 새로 받는다.
        • •DTO → Option 변환. API 응답 필드(univCode/aiBdPart/ majorIdHsb)를 드롭다운용 { value, label } 로 hook 에서 매핑한다.
        • •univCode 보관 + 단계별 로딩 + race 가드. 학과 API 가 univCode 를 다시 요구하고, fetch 가 여러 곳이라 로딩이 단계마다 필요하며, 빠른 재선택 시 요청 번호(useRef)로 마지막 응답만 반영한다.
