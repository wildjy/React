Cascade Dropdown 학습 샘플

대학 → 계열 → 학과 순서로 상위 선택이 하위 옵션을 결정하는 연쇄 의존 드롭다운 패턴입니다. 왼쪽에서 직접 선택해 보고, 오른쪽에서 내부 상태와 캐시가 어떻게 변하는지 실시간으로 확인하세요.

POINT 01

IoC (제어의 역전)

함수를 prop / return 값으로 주입해서 hook 의 동작을 외부에서 결정. fetcher 만 갈아끼우면 대학·고교·지역 등 어디든 같은 hook 재사용.

const onUniversityChange = async (id) => {...}
POINT 02

Cascade (연쇄 의존)

상위 선택값이 하위 옵션을 결정. 상위가 바뀌면 하위 선택값·옵션을 반드시 리셋해야 잘못된 조합이 남지 않음.

setSelected({ ..., majorTypeCode: "", majorId: "" })
POINT 03

상태 vs 캐시

화면에 그릴 옵션은 useState (리렌더 필요), 내부 계산용 raw 데이터는 useRef (리렌더 불필요). 역할이 다름.

useRef<Majors[]>([]) // 화면에 직접 안 그림

④ 동작 흐름 — 단계별로 무엇이 일어나는가

왼쪽 데모를 조작할 때마다 이 순서대로 함수가 실행됩니다. 동작 로그와 비교해 보세요.

1

대학 변경 → onUniversityChange

  1. 1.setSelected({ universityId, majorTypeCode: "", majorId: "" }) — 하위 cascade 리셋
  2. 2.setMajorOptions([]) / setMajorTypeOptions([]) — 옵션 비움
  3. 3.await mockFetchUnivMajors(id) — 가짜 서버 호출 (300ms 지연)
  4. 4.cachedMajorsRef.current = majors — 응답을 캐시에 저장 (리렌더 X)
  5. 5.majors → majorTypeCode set 추출 → ALL_MAJOR_TYPES 와 교집합 → setMajorTypeOptions
2

계열 변경 → onMajorTypeChange

  1. 1.setSelected(prev → { ...prev, majorTypeCode, majorId: "" }) — 학과만 리셋
  2. 2.서버 호출 없음 — cachedMajorsRef.current 를 majorTypeCode 로 filter
  3. 3.filter 결과를 { value, label } 로 변환 → setMajorOptions
3

학과 변경 → onMajorChange

  1. 1.하위 cascade 없음 — setSelected(prev → { ...prev, majorId }) 만 실행
  2. 2.이후 폼 제출·서버 전송 등에서 selected 객체를 통째로 사용

⑤ 왜 useState 와 useRef 를 나눠 썼나?

같은 "데이터 보관"이라도 화면에 영향을 주는지에 따라 도구가 달라집니다.

useState · majorTypeOptions / majorOptions

값이 바뀌면 화면을 다시 그려야 함.<select>안의 옵션 목록이 그것.

→ setState 가 트리거하는 리렌더가 필수.

useRef · cachedMajorsRef

값이 바뀌어도 화면이 바뀌지 않음. 다음 계열 변경 때 filter 입력으로만 쓰는 raw 데이터.

→ useState 로 두면 불필요한 리렌더 1회 추가. useRef 가 더 정확.

판단 기준: "이 값이 바뀌었을 때 화면 어딘가가 달라져야 하나?" → Yes 면 useState, No 면 useRef. 캐시·이전값 기록·외부 객체 참조 등은 보통 useRef.

⑥ 다음 단계 — 실무에서 IoC 로 확장

지금은 hook 안에 mockFetchUnivMajors 가 박혀 있습니다. 실무 코드는 fetcher 를 prop 으로 받아서 hook 을 여러 도메인(대학·고교·지역)에 재사용할 수 있게 분리합니다.

// Before — fetcher 가 hook 안에 박혀 있음 (학습용)
function useCascadeDropdowns() {
  const majors = await mockFetchUnivMajors(id);
  ...
}

// After — fetcher 를 prop 으로 주입 (실무용)
function useCascadeDropdowns(fetcher: FetchMajorTypesByUniv) {
  const majors = await fetcher(id);   // 호출부만 바꿔치기 가능
  ...
}

// 사용처: 대학용·고교용·지역용 fetcher 만 갈아끼우면 같은 hook 재사용
